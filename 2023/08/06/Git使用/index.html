基础使用

图形化Git管理工具：Sourcetree，SmartGit等

拉取与提交

  • Pull（拉取同步远程仓库） -> Commit（提交本地仓库）-> Push（本地代码推送到远程仓库）

  • 先拉取再提交，提交不成功小事，有概率蜜汁报错甚至需要回退

  • 补充用户名和邮箱

解决冲突

  • 合并

  • 看提交记录，ours 自己本地提交冲突的，theirs 当前主干提交的（要合并的分支），选择需要合并的主干进行合并

    • 保存需要保留的版本

    • 驻藏本地

    • 提交

      From：https://blog.csdn.net/james_xiaojun/article/details/100572359?utm_medium=distribute.pc_relevant.none-task-blog-2~default~baidujs_baidulandingword~default-0-100572359-blog-102611114.235

  • 代码可以丢弃自己，通过ide可以恢复

版本管理

  • 每封一个版本产生一个新分支，开发基本是在主干上开发
  • Branches上切换分支

其他功能

切换账户

SourceTree 更换用户名称和密码

坑

git反复弹登录

  • 加本地用户凭证：git config --global credential.helper store
